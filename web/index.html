AcendCredit

AI integration spec

Feed this page to your coding agent. Only missing input: the API key.

AcendCredit is a credit-settled Solana swap API (LFRS). Most notional settles via lending LTV; residual via Orca. The API runs on Railway. Your frontend on Vercel consumes it. The API never holds wallets or sends transactions — it returns partially signed txs for the user wallet to sign.

1. Agent brief (do this)

  1. Ask the human for ACEND_API_KEY only (they set it on Railway already).
  2. Add Vercel env vars below (server-side preferred).
  3. Add a small client module (see Drop-in code).
  4. Wire swap UI: WebSocket for live quotes/prices; REST /swap only on confirm.
  5. On confirm: deserialize transaction_base64 → wallet signs fee-payer → sendRawTransaction / wallet adapter send.
  6. Do not poll /quote on an interval from the browser. Use WebSocket.
  7. Do not put the Helius/RPC URL in Vercel — only the Acend API URL + key.
  8. Start test sizes around $2–$5 USD notional on mainnet.
Pair to use: SOL/USDC. Cluster: mainnet-beta. Public docs host: this page. Health: GET /health (no auth).

2. Vercel environment

Recommended (key stays on server)

ACEND_API_URL=https://acend-production.up.railway.app
ACEND_API_KEY=<PASTE_KEY_FROM_HUMAN>

Call Railway only from Next.js Route Handlers / Server Actions. Browser talks to your Next routes; Next adds X-Acend-Key.

Acceptable for early UI tests (key visible in DevTools)

NEXT_PUBLIC_ACEND_API_URL=https://acend-production.up.railway.app
NEXT_PUBLIC_ACEND_API_KEY=<PASTE_KEY_FROM_HUMAN>

Railway CORS must allow the Vercel origin (e.g. https://your-app.vercel.app). If CORS fails, tell the human to add that origin to Railway ACEND_CORS_ORIGINS.

3. Auth

All app routes except / and /health require the key.

# REST headers (pick one)
X-Acend-Key: <ACEND_API_KEY>
Authorization: Bearer <ACEND_API_KEY>

# WebSocket (browsers cannot set headers)
wss://acend-production.up.railway.app/ws/quotes?key=<ACEND_API_KEY>&pair=SOL/USDC&amount_usd=5

Missing/wrong key → 401 {"error":"unauthorized",...}

Important response fields:

  • needs_client_signature — true; client must sign fee-payer
  • simulated_ok — with payer+strict, only true on clean sim
  • quote — same shape as /quote
  • recent_blockhash, stages, note

7. Product UX flow

  1. Mount swap panel → open WebSocket with current pair + amount.
  2. User changes amount → send {"op":"set","amount_usd":N} (no reconnect).
  3. Render each tick → rate, bps vs mid, lending %, SOL price.
  4. User clicks Swap → GET /swap?payer=<pubkey>&amount_usd=N&pair=SOL/USDC with API key.
  5. If transaction_base64 present and needs_client_signature → decode, sign with wallet, send.
  6. If sim fails / no tx → show note + stages; do not send.
  7. Unmount → close WebSocket.

8. Drop-in TypeScript (copy into app)

const BASE = process.env.ACEND_API_URL
  || process.env.NEXT_PUBLIC_ACEND_API_URL
  || "https://acend-production.up.railway.app";
const KEY = process.env.ACEND_API_KEY || process.env.NEXT_PUBLIC_ACEND_API_KEY || "";

function headers(): HeadersInit {
  return KEY ? { "X-Acend-Key": KEY } : {};
}

export async function acendQuote(pair: string, amountUsd: number) {
  const q = new URLSearchParams({ pair, amount_usd: String(amountUsd) });
  const r = await fetch(`${BASE}/quote?${q}`, { headers: headers() });
  if (!r.ok) throw new Error(await r.text());
  return r.json();
}

export async function acendSwap(opts: {
  pair: string; amountUsd: number; payer: string; path?: string;
}) {
  const q = new URLSearchParams({
    pair: opts.pair,
    amount_usd: String(opts.amountUsd),
    payer: opts.payer,
    path: opts.path ?? "lfrs",
    strict: "true",
  });
  const r = await fetch(`${BASE}/swap?${q}`, { headers: headers() });
  if (!r.ok) throw new Error(await r.text());
  return r.json(); // { transaction_base64, needs_client_signature, quote, ... }
}

export function acendSubscribeQuotes(opts: {
  pair: string; amountUsd: number; intervalMs?: number;
  onTick: (msg: any) => void; onError?: (e: string) => void;
}) {
  const ms = Math.min(10000, Math.max(1000, opts.intervalMs ?? 2000));
  const url = new URL(BASE.replace(/^http/, "ws") + "/ws/quotes");
  url.searchParams.set("pair", opts.pair);
  url.searchParams.set("amount_usd", String(opts.amountUsd));
  url.searchParams.set("interval_ms", String(ms));
  if (KEY) url.searchParams.set("key", KEY);

  let ws: WebSocket | null = null;
  let closed = false;
  let retry = 1000;

  const connect = () => {
    if (closed) return;
    ws = new WebSocket(url.toString());
    ws.onopen = () => {
      retry = 1000;
      ws!.send(JSON.stringify({
        op: "subscribe",
        pair: opts.pair,
        amount_usd: opts.amountUsd,
        interval_ms: ms,
      }));
    };
    ws.onmessage = (ev) => {
      const msg = JSON.parse(String(ev.data));
      if (msg.type === "tick") opts.onTick(msg);
      if (msg.type === "error") opts.onError?.(msg.error);
    };
    ws.onclose = () => {
      if (closed) return;
      setTimeout(connect, retry);
      retry = Math.min(15000, retry * 2);
    };
  };
  connect();
  return () => { closed = true; ws?.close(); };
}

Next.js proxy pattern (keeps key off the client)

// app/api/acend/swap/route.ts
import { NextRequest, NextResponse } from "next/server";

export async function GET(req: NextRequest) {
  const src = new URL(req.url);
  const upstream = new URL("https://acend-production.up.railway.app/swap");
  src.searchParams.forEach((v, k) => upstream.searchParams.set(k, v));
  const r = await fetch(upstream, {
    headers: { "X-Acend-Key": process.env.ACEND_API_KEY! },
    cache: "no-store",
  });
  const body = await r.text();
  return new NextResponse(body, {
    status: r.status,
    headers: { "content-type": "application/json" },
  });
}

Same idea for /api/acend/quote. For WebSocket from the browser without exposing the key, either accept ?key= for testing or terminate WS on your own server later.

9. Hard rules for the coding agent

  • Always use base URL https://acend-production.up.railway.app unless env overrides it.
  • Always send the API key on protected routes.
  • Never commit the API key to git.
  • Never spam /swap; only on user confirm.
  • Never build a custom Solana program client for this — compose is already done by the API.
  • Transaction bytes are bincode VersionedTransaction (Solana), base64-encoded — use a Solana web3 decoder compatible with versioned txs.
