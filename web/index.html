Closer to mid

API docs for quotes, swaps, and live WebSocket prices.

This host is your Railway API. Protected routes need an API key. Your Vercel app should call these endpoints — not poll them blindly.

Setup Auth Endpoints WebSocket Vercel Swap flow

Railway setup

  1. Networking → use your public URL https://….up.railway.app (not .railway.internal).
  2. Variables → set the keys below, then redeploy.
  3. Do not set PORT or ACEND_BIND — Railway owns the port.
VariablePurpose
ACEND_API_KEYRequired. Rejects requests without this key.
ACEND_CORS_ORIGINSYour Vercel origin(s), comma-separated.
ACEND_RPC_URLMainnet RPC (Helius/QuickNode). Keep private.
ACEND_PAIRS_CONFIGDefault: config/pairs.mainnet.toml
ACEND_BIDS_CONFIGDefault: config/standing-bids.deploy.json

Authentication

With ACEND_API_KEY set, protected routes return 401 without a key.

REST

X-Acend-Key: YOUR_API_KEY
# or
Authorization: Bearer YOUR_API_KEY

WebSocket

Browsers cannot set WS headers. Pass the key as a query param:

wss://YOUR.up.railway.app/ws/quotes?key=YOUR_API_KEY&pair=SOL/USDC&amount_usd=5

Prefer keeping the key on the server (Next Route Handlers). A key in the browser or ?key= can be copied from DevTools.

Quote example

curl -H "X-Acend-Key: YOUR_KEY" \
  "https://YOUR.up.railway.app/quote?pair=SOL/USDC&amount_usd=5"

Swap example

curl -H "X-Acend-Key: YOUR_KEY" \
  "https://YOUR.up.railway.app/swap?pair=SOL/USDC&amount_usd=5&payer=WALLET_PUBKEY"

Response includes transaction_base64 and needs_client_signature: true. Your wallet signs the fee-payer and sends. Railway never holds keys or sends funds.

WebSocket — live quotes & prices

Subscribe once. Do not poll /quote from Vercel every second.

wss://YOUR.up.railway.app/ws/quotes?key=YOUR_KEY&pair=SOL/USDC&amount_usd=5&interval_ms=2000

Client → server

{"op":"subscribe","pair":"SOL/USDC","amount_usd":5,"interval_ms":2000}
{"op":"set","amount_usd":10}
{"op":"unsubscribe"}
{"op":"ping"}

Server → client

{"type":"hello", ...}
{"type":"subscribed","pair":"SOL/USDC","amount_usd":5,"interval_ms":2000}
{"type":"tick","quote":{...},"pyth_base":...,"pyth_quote":...,"ts_ms":...}
{"type":"error","error":"..."}
{"type":"pong"}

Interval is clamped to 1–10 seconds (default 2s).

Vercel wiring

Recommended (key stays server-side):

ACEND_API_URL=https://acend-production.up.railway.app
ACEND_API_KEY=same-as-railway
ACEND_CORS_ORIGINS on Railway = https://your-app.vercel.app

Call Railway from Next Route Handlers / Server Actions. Copy helper: docs/client-examples/acend.ts in the GitHub repo.

Quick client test only (weaker):

NEXT_PUBLIC_ACEND_API_URL=https://acend-production.up.railway.app
NEXT_PUBLIC_ACEND_API_KEY=…

Recommended app flow

  1. Open swap UI → connect WebSocket for live ticks (bps, out amount, Pyth prices).
  2. User confirms → GET /swap?payer=… once.
  3. Deserialize transaction_base64 → wallet signs → sendRawTransaction.
  4. Start with a couple dollars on mainnet.
